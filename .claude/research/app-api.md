# Directory `app/api/`

## Overview

This directory contains Otoroshi's **generic REST API framework**. It provides an abstraction to handle CRUD operations on all entities uniformly.

## Files

### `api.scala` (~900 lines)
**Role**: Generic CRUD framework for the administration REST API.

**Main classes**:

#### Resource Model
- `Resource` - Definition of a resource exposed via the API
  - `kind`: Resource type (e.g., "Route", "ApiKey")
  - `pluralName`: Plural name for URLs (e.g., "routes")
  - `singularName`: Singular name
  - `group`: API group (e.g., "proxy.otoroshi.io")
  - `version`: API version (ResourceVersion)
  - `access`: CRUD operations implementation

- `ResourceVersion` - API version management
  - `name`: Version name (e.g., "v1")
  - `served`: Whether this version is served
  - `deprecated`: Whether this version is deprecated
  - `storage`: Whether this is the storage version
  - `schema`: Optional JSON Schema (otherwise generated by reflection)

#### CRUD Abstraction
- `ResourceAccessApi[T]` - Trait defining operations on a resource
  - `clazz`: Entity class
  - `format`: Play JSON Format for serialization
  - `key(id)`: Generates storage key
  - `extractId(value)`: Extracts ID from entity
  - `template()`: Returns a JSON template
  - Flags: `canRead`, `canCreate`, `canUpdate`, `canDelete`, `canBulk`
  - `writeValidation()`: Custom write validation
  - `deleteValidation()`: Custom delete validation

#### Actions
- `WriteAction` - Enum: `Create` | `Update`
- `DeleteAction` - Enum: `DeleteOne` | `DeleteAll`

**Features**:
- Uniform CRUD pattern for all entities
- Bulk operations support (mass create/update/delete)
- Validation via JSON Schema (auto-generated by Scala reflection)
- Template generation for each entity
- Pagination and result filtering
- JSON Patch support (RFC 6902) for partial updates
- Automatic modification audit
- User rights management (read/write)

### `openapi.scala` (~1030 lines)
**Role**: Automatic OpenAPI 3.0 specification generation.

**Main classes**:
- `OpenApi` - Singleton object to generate OpenAPI document

**Generation methods**:
- `buildCount(resource)`: `/_count` endpoint
- `buildById(resource)`: `/{id}` endpoints (GET, PUT, PATCH, DELETE)
- `buildTemplate(resource)`: `/_template` endpoint
- `buildBulk(resource)`: `/_bulk` endpoint
- `buildResource(resource)`: Base endpoints (GET all, POST)
- `buildPaths(resource)`: Generates all paths for a resource
- `cleanupSchemas()`: Cleans and normalizes schemas
- `generate()`: Generates the complete OpenAPI document

**Features**:
- Automatic generation from resource definitions
- Generated document caching
- Additional schemas support (JSON files)
- Filtering by extension group
- OpenAPI 3.0.3 format
- Security: Basic Auth (otoroshi_auth)

**Generated URL structure**:
```
/apis/{group}/{version}/{pluralName}           # GET all, POST
/apis/{group}/{version}/{pluralName}/_count    # GET count
/apis/{group}/{version}/{pluralName}/_template # GET template
/apis/{group}/{version}/{pluralName}/_bulk     # POST/PUT/PATCH/DELETE bulk
/apis/{group}/{version}/{pluralName}/{id}      # GET/PUT/PATCH/DELETE by id
```

## Relationships with Other Components

```
api/
    │
    ├── Used by → controllers/adminapi/*.scala (all CRUD endpoints)
    ├── Used by → controllers/SwaggerController.scala (exposes /api/openapi.json)
    │
    ├── Depends on → models/*.scala (entity definitions)
    ├── Depends on → storage/ (persistence)
    └── Depends on → actions/api.scala (authentication)
```

## Key Points

1. **DRY pattern**: Single CRUD implementation for all entities
2. **Auto-documentation**: OpenAPI automatically generated
3. **Validation**: JSON Schema generated by Scala reflection
4. **Bulk operations**: Native support for mass operations (ndjson)
5. **Versioning**: API versioning support (v1, v2, etc.)
6. **Extensible**: Extensions can add their own resources

## Usage Example

```scala
// Define a resource
val routeResource = Resource(
  kind = "Route",
  pluralName = "routes",
  singularName = "route",
  group = "proxy.otoroshi.io",
  version = ResourceVersion("v1", served = true, deprecated = false, storage = true),
  access = new RouteAccessApi()
)

// REST API is automatically available:
// GET    /apis/proxy.otoroshi.io/v1/routes
// POST   /apis/proxy.otoroshi.io/v1/routes
// GET    /apis/proxy.otoroshi.io/v1/routes/{id}
// PUT    /apis/proxy.otoroshi.io/v1/routes/{id}
// DELETE /apis/proxy.otoroshi.io/v1/routes/{id}
// etc.
```
